## Problem

`app/api/contact/route.ts` fires n8n webhooks in parallel with HubSpot
contact creation, passing `contactId: ""` (empty string). This means
**SYS 01 — Website Form Lead Intake** in the n8n workflows repo receives
an empty contactId, hits the guard node, and exits immediately. The
entire downstream pipeline — deal creation, `deal_pain_points` population,
and SYS 02 outreach enrollment — never fires.

## Root Cause

`n8nPayload` is built before the HubSpot Contacts API call resolves,
so `contactId` is always an empty string at the time n8n is called.

## What Needs to Change

1. **Execution order** — HubSpot contact creation must fully resolve and
   return a real `contactId` before n8n fires
2. **409 handling** — when a contact already exists and we PATCH them,
   capture the PATCH result's `id` and still fire n8n (existing contacts
   with new pain points need a new deal)
3. **Tier classification** — call Claude API (using `fetch`, not the SDK)
   to classify `painPoint` as `starter` or `growth` before firing n8n.
   Falls back to `starter` on any error. Uses `claude-haiku-4-5-20251001`
   for speed and cost
4. **Env var cleanup** — remove `N8N_LEAD_SCORING_WEBHOOK_URL`. Keep
   `N8N_CONTACT_WEBHOOK_URL` as the single n8n target pointing to
   `/webhook/hubspot-website-form-lead`
5. **n8n payload** — fire with:
```json{
"contactId": "<real_id>",
"createDeal": true,
"dealStage": "qualifiedtobuy",
"tier": "starter|growth",
"painOverride": ""
}
```

## Acceptance Criteria

- [ ] Form submission creates/updates HubSpot contact and captures real `id`
- [ ] Claude classifies `painPoint` as `starter` or `growth` before n8n fires
- [ ] n8n webhook fires with non-empty `contactId` and correct `tier`
- [ ] 409 conflict (existing contact) still fires n8n with the existing `id`
- [ ] HubSpot API error does not return 500 to the user — logs and continues
- [ ] Claude API error falls back to `starter` silently — never breaks form
- [ ] `N8N_LEAD_SCORING_WEBHOOK_URL` is removed from all references
- [ ] `CONTACT_INTEGRATION_DEBUG=true` exposes `n8nFired`, `tier`, `contactAction`
- [ ] No changes to form schema, reCAPTCHA, Resend, or HubSpot Forms blocks
- [ ] Verified in Vercel logs: `[n8n] Firing webhook for` log appears with real contactId
